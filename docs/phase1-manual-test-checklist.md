## 4. 新增（Add）

### 4-1. 必填欄位驗證（對應 #10）

- [ ] **record_date（記錄日期）** 清空後按儲存，顯示繁中錯誤提示（例如：「記錄日期為必填」）
- [ ] **location（地點）** 清空後按儲存，顯示繁中錯誤提示（例如：「地點為必填」）
- [ ] **machine_no（機號）** 清空後按儲存，顯示繁中錯誤提示（例如：「機號為必填」）

### 4-2. 日期格式與有效日期驗證（對應 #10）

- [ ] `record_date` 輸入格式錯誤（例如 `2024/02/01` 或 `abcdefgh`），按儲存顯示「記錄日期格式錯誤」
- [ ] `record_date` 輸入不存在的日期（例如 `2024-02-30`），按儲存顯示「記錄日期不是有效日期」
- [ ] `record_date` 輸入合法日期（例如 `2024-02-29`，2024 為閏年），按儲存**不**顯示日期錯誤
- [ ] 初次進入新增頁，`record_date` 預設為今天（`YYYY-MM-DD`）
- [ ] 按下「清除」/「重置」後，`record_date` 重新回到今天

### 4-3. 機號等欄位英數字點驗證（對應 #11）

規則：允許英文字母、數字、英文句點（`.`）；**不允許空白、連字號（`-`）、中文或其他符號**。  
適用欄位：`machine_no`、`vfd_model`（升降機系統）、`vfd_software`（升降機軟件）、`motor_model`（摩打型號）

- [ ] `machine_no` 輸入 `ABC123.01`，按儲存**不**顯示格式錯誤
- [ ] `machine_no` 輸入 `TC-01`（含 `-`），按儲存顯示「只能輸入英文/數字/句點(.)」
- [ ] `machine_no` 輸入 `A B`（含空白），按儲存顯示格式錯誤
- [ ] `vfd_model` 輸入 `TestA.01`，按儲存**不**顯示格式錯誤（選填欄位有值時驗證）
- [ ] `vfd_model` 輸入 `Test-A`，按儲存顯示格式錯誤
- [ ] `vfd_software` / `motor_model` 同上規則，驗證各一次

### 4-4. 故障碼純數字 + 數字鍵盤（對應 #12）

- [ ] 點擊 `fault_code`（故障碼）欄位，鍵盤自動切換為數字鍵盤
- [ ] `fault_code` 為選填，留空可正常儲存（不顯示錯誤）
- [ ] `fault_code` 輸入純數字（例如 `404`），按儲存**不**顯示格式錯誤
- [ ] `fault_code` 輸入含字母（例如 `E04`），按儲存顯示「故障碼只能輸入數字」

### 4-5. 儲存成功提示（對應 #13）

- [ ] 所有必填欄位填寫正確後按儲存，顯示提示（例如 Alert：「尚未接 DB，已通過驗證」或儲存成功訊息）

## 9. 常見問題 / 排除

### Q1：資料庫沒有資料，怎麼建立測試資料？
1. 進入「新增」Tab，手動新增至少 3 筆資料（不同地點、機號）。
2. （若有 seed 功能）參考 `db/` 目錄下的 seed 腳本，或執行 Issue #23 完成的「一鍵塞 10 筆測試資料」功能。

### Q2：Expo Go 顯示舊畫面 / 改了 code 沒更新？
1. 在 Metro 終端機按 `r` 強制 reload。
2. 若仍未更新，停止 Metro 後執行 `npx expo start -c`（清除快取）。
3. 在 Expo Go App 中長按螢幕，選「Reload」。

### Q3：iOS 權限提示（相機、檔案存取）
- 匯出功能首次執行時，iOS 可能要求「允許存取媒體庫 / 檔案」。
- 請選「允許」；若不小心選「拒絕」，到「設定 → 隱私 → 媒體與 Apple Music」中重新開啟。

### Q4：安裝 npm 套件報錯？
1. 先刪除 `node_modules` 資料夾：`rm -rf node_modules`。
2. 重新執行 `npm install`。
3. 若仍有問題，確認 Node.js 版本為 **20.x**（`node -v`）。
